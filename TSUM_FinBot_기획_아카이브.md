## 3. 우리 프로젝트 결정사항

### 도메인
**재무 — 기업 실적보고서 요약 챗봇 (v1) → 이상탐지 (v2)**

선택 이유:
- DART 공개 데이터 풍부 → 실제 데이터 수집 가능
- GPT API로 합성 Q&A 생성에 최적 (멘토 스택과 일치)
- 기업이 실제로 필요로 하는 도메인
- 수치 기반이라 이상탐지로 자연스럽게 확장 가능

### 모델 선택 (멘토 의도 추측)
- 멘토가 "중국쪽 사용예정" → **DeepSeek-R1 or DeepSeek-V2** 계열로 추정
- 한국어 지원 필요시 EXAONE(LG) 대안
- **우선 DeepSeek 계열로 진행, 멘토 확정 시 교체**

### 파인튜닝 방식 (멘토 정보 없음 → 추측)
- 코랩 무료 GPU 한계상 **LoRA (PEFT)** 방식 사용
- 멘토도 코랩 쓴다고 했으니 동일 방식일 가능성 높음

### 데이터 규모
- 예습이 목적이므로 규모 불필요
- **최소한으로**: 보고서 10개사 × GPT Q&A 50쌍 = 500개면 충분

## 4. 전체 파이프라인

```text
[1단계] 데이터 수집
└── DART API → 코스피 대형주 10~20개사 분기 실적보고서 수집
└── API 키: DART (보유)

[2단계] 전처리
└── PDF 텍스트 추출
└── 섹션 분리: 매출 / 영업이익 / 순이익 / 전망 등

[3단계] 합성 데이터 생성
└── GPT API (gpt-4o-mini 사용, 비용 절감)
└── 각 보고서 → Q&A 쌍 자동 생성
└── 예시: Q."삼성전자 23Q4 영업이익?" A."xxx억원, 전분기 대비 xx% 감소"
└── 목표: ~500개 Q&A 쌍
└── API 키: OpenAI (보유, $10 한도 설정 완료)

[4단계] 파인튜닝
└── 환경: Google Colab (무료 GPU)
└── 모델: DeepSeek 계열 (허깅페이스에서 로드)
└── 방식: LoRA (PEFT) — 코랩 메모리 한계 대응
└── 라이브러리: transformers, peft, trl

[5단계] 서빙 & 테스트
└── 코랩에서 모델 로드 후 질문 → 답변 확인
└── (멘토 확정 후) RunPod으로 이전
└── 결과 캡처 + 아카이빙
```
